# Contributing to Tech Books Digest

Thank you for your interest in contributing!  
This project is a collaborative effort to read, summarise, and understand technical books.

We welcome:
- Chapter summaries  
- New book additions  
- Corrections and improvements  
- Diagrams, examples, and explanations  
- Insights and notes

## ğŸ§­ How to Contribute

### 1. Fork the repository  
Click the **Fork** button in the top-right corner of GitHub.

### 2. Create a new branch  
Use a meaningful branch name:

shashaank/ddia-ch01  
friend/clean-code-ch02  

### 3. Add or edit notes  
Add book folders, chapters, summaries, or diagrams.  
Follow templates inside `/templates`.

### 4. Commit your changes  
Use descriptive messages:

- Add chapter 01 notes for DDIA  
- Add diagram for event sourcing flow  
- Update glossary with CAP theorem  
- Improve consistency in chapter summaries  

### 5. Open a Pull Request  
In your PR, include:
- Book name  
- Chapter number (if applicable)  
- Summary of changes  

### 6. Get a review  
One of the maintainers will review, suggest improvements, and merge.

## ğŸ“ Style Guidelines

- Use Markdown with proper headings.  
- Keep notes concise but clear.  
- Add diagrams whenever helpful (Mermaid or images).

- Add interview-relevant questions to `shared/interview-questions.md`.  
- Keep file names lowercase and hyphen-separated.
